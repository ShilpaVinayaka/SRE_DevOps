## ✅ Coding Question #6 — Monitor Logs for Errors (Very Common SRE Question)

### Question

Write a **shell command/script** that:

* Continuously monitors a log file
* If the word **"ERROR"** appears
* Print an **alert message**

Example log file:

```text
/var/log/app.log
```

---

## 💡 What the interviewer is testing

They want to see if you know:

* `tail`
* `grep`
* log monitoring
* real-time debugging

This is **very common in production troubleshooting**.

---

## 🧪 Simple Command Solution

```bash
tail -f /var/log/app.log | grep "ERROR"
```

---

## 🔎 Explanation

| Command        | Purpose                               |
| -------------- | ------------------------------------- |
| `tail -f`      | Continuously monitors new log entries |
| `grep "ERROR"` | Filters lines containing "ERROR"      |

So whenever an error appears, it will immediately display it.

---

## Example Output

```text
2026-03-11 14:20:31 ERROR Database connection failed
```

---

## 🧪 Better Script Version (Interview-Ready)

```bash
#!/bin/bash

logfile="/var/log/app.log"

tail -F $logfile | while read line
do
    if [[ $line == *"ERROR"* ]]
    then
        echo "ALERT: Error detected -> $line"
    fi
done
```

---

## Example Output

```
ALERT: Error detected -> 2026-03-11 ERROR DB connection timeout
```

---

## ⭐ Follow-Up Questions Interviewers Ask

### ❓ How to count total errors?

```bash
grep "ERROR" app.log | wc -l
```

---

### ❓ How to monitor only the last 100 lines?

```bash
tail -100 app.log
```

---

### ❓ How to search errors from yesterday?

```bash
grep "2026-03-10" app.log | grep ERROR
```
