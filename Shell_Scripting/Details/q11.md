## ✅ Coding Question #11 — Extract Fields from a Log File

### Question

You have a **web server log file** like this:

```text
192.168.1.10 - - [10/Mar/2026:10:15:22] "GET /api/users HTTP/1.1" 200
10.0.0.2 - - [10/Mar/2026:10:16:02] "POST /login HTTP/1.1" 401
```

Write commands to extract:

1️⃣ **IP addresses**
2️⃣ **HTTP endpoints**
3️⃣ **HTTP status codes**

---

## 💡 What the interviewer is testing

They want to see if you understand:

* `awk`
* column extraction
* log structure
* quick data parsing

Logs are usually **space-separated fields**, so `awk` is perfect.

---

# 🧪 Extract IP Addresses

```bash
awk '{print $1}' access.log
```

Output:

```
192.168.1.10
10.0.0.2
```

`$1` = first column (IP address)

---

# 🧪 Extract HTTP Endpoint

```bash
awk '{print $7}' access.log
```

Output:

```
/api/users
/login
```

`$7` = endpoint path.

---

# 🧪 Extract Status Code

```bash
awk '{print $9}' access.log
```

Output:

```
200
401
```

`$9` = HTTP status code.

---

# 🧪 Extract Only Failed Requests (status ≥ 400)

```bash
awk '$9 >= 400 {print $0}' access.log
```

Output:

```
10.0.0.2 - - [10/Mar/2026:10:16:02] "POST /login HTTP/1.1" 401
```

---

# ⭐ Follow-Up Questions Interviewers Ask

### ❓ Count number of 500 errors

```bash
awk '$9 == 500' access.log | wc -l
```

---

### ❓ Find most requested endpoint

```bash
awk '{print $7}' access.log | sort | uniq -c | sort -nr | head
```

---

### ❓ Find requests from a specific IP

```bash
grep "192.168.1.10" access.log
```

---

# 🧠 Real SRE Insight

Log parsing like this helps identify:

* **traffic spikes**
* **failing APIs**
* **security attacks**
* **performance bottlenecks**

Tools like **ELK, Splunk, and Datadog logs** internally do similar parsing.
