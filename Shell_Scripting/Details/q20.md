## ✅ Coding Question #20 — Check if a Port is Open on a Server

### Question

Write a **shell command/script** to:

* Check if a **specific port is open**
* On a **given server**
* Print whether the port is **OPEN or CLOSED**

Example:

```text
Server: google.com
Port: 80
```

---

## 💡 What the interviewer is testing

They want to see if you know:

* `nc` (netcat) or `telnet`
* networking basics
* port connectivity checks

This is very useful when debugging:

* firewall issues
* service availability
* network connectivity

---

# 🧪 Simple Command Using `nc`

```bash
nc -zv google.com 80
```

---

## 🔎 Explanation

| Option | Meaning                        |
| ------ | ------------------------------ |
| `nc`   | Netcat networking tool         |
| `-z`   | Scan port without sending data |
| `-v`   | Verbose output                 |

---

## Example Output

Open port:

```
Connection to google.com 80 port [tcp/http] succeeded!
```

Closed port:

```
nc: connect to google.com port 80 (tcp) failed
```

---

# 🧪 Script Version

```bash
#!/bin/bash

host="google.com"
port=80

if nc -z $host $port
then
    echo "Port $port is OPEN on $host"
else
    echo "Port $port is CLOSED on $host"
fi
```

---

## Example Output

```
Port 80 is OPEN on google.com
```

or

```
Port 80 is CLOSED on google.com
```

---

# ⭐ Alternative Methods

### Using `telnet`

```bash
telnet google.com 80
```

---

### Using `curl`

```bash
curl google.com:80
```

---

### Using `nmap`

```bash
nmap -p 80 google.com
```

## ⭐ Follow-up Questions Interviewers Ask

```bash
ss -tuln
```

or

```bash
netstat -tuln
```

---

### ❓ Check which process is using a port

```bash
lsof -i :8080
```
