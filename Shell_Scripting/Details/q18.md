## ✅ Coding Question #18 — Monitor Memory Usage and Alert

### Question

Write a **shell script** that:

* Checks **memory usage**
* If memory usage is **greater than 80%**
* Print an **alert message**

---

## 💡 What the interviewer is testing

They want to see if you know:

* `free`
* parsing command output
* arithmetic in shell
* system monitoring basics

This is a **very common SRE monitoring task**.

---

## 🧪 Step 1 — Check Memory Usage

```bash
free -h
```

Example output:

```
              total        used        free
Mem:           16Gi       10Gi        6Gi
```

But for scripting we usually use:

```bash
free
```

---

## 🧪 Script Solution

```bash
#!/bin/bash

threshold=80

usage=$(free | awk '/Mem/ {printf("%.0f"), $3/$2 * 100}')

if [ $usage -gt $threshold ]
then
    echo "ALERT: Memory usage is ${usage}%"
else
    echo "Memory usage normal: ${usage}%"
fi
```

---

## 🔎 Explanation

| Part        | Meaning              |
| ----------- | -------------------- |
| `free`      | Shows memory usage   |
| `/Mem/`     | Selects memory row   |
| `$3`        | Used memory          |
| `$2`        | Total memory         |
| `$3/$2*100` | Calculate percentage |

---

## Example Output

High memory:

```
ALERT: Memory usage is 85%
```

Normal memory:

```
Memory usage normal: 45%
```

---

## ⭐ Follow-up Questions Interviewers Ask

### ❓ How to check memory usage interactively?

```bash
top
```

or

```bash
htop
```

---

### ❓ How to see memory usage per process?

```bash
ps aux --sort=-%mem | head
```

---

### ❓ How to see swap usage?

```bash
free -h
```

Look at the **Swap** row.

---

## 🧠 Real SRE Insight

High memory usage usually happens because of:

* memory leaks
* too many containers
* large caches
* runaway processes
* JVM heap issues

Monitoring systems like:

* Prometheus
* Datadog
* CloudWatch
* Nagios

basically implement **similar logic internally**.
