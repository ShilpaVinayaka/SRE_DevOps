## ✅ Coding Question #2 — Log Analysis (Very Common)

### Question

You have an **application log file**:

```text
/var/log/app.log
```

Write a **shell command/script** to **find the top 5 IP addresses that made the most requests**.

Example log line:

```text
192.168.1.10 - - [10/Mar/2026:10:15:22] "GET /api/users HTTP/1.1" 200
```

---

## 💡 What the interviewer wants to test

* `awk`
* `sort`
* `uniq`
* `pipe usage`
* Understanding **log processing**

This is **extremely common for SRE roles** because logs are analyzed like this in production.

---

## 🧠 Step-by-Step Logic

1. **Extract the IP address** → first column
2. **Sort them**
3. **Count occurrences**
4. **Sort by count**
5. **Show top 5**

---

## 🧪 Interview-Ready Solution

```bash
awk '{print $1}' /var/log/app.log | sort | uniq -c | sort -nr | head -5
```

---

## 🔎 Breakdown

| Command            | Purpose                              |
| ------------------ | ------------------------------------ |
| `awk '{print $1}'` | Extract first column (IP address)    |
| `sort`             | Group identical IPs together         |
| `uniq -c`          | Count occurrences                    |
| `sort -nr`         | Sort numerically in descending order |
| `head -5`          | Show top 5                           |

---

## 📌 Example Output

```text
120 192.168.1.10
95 10.10.1.2
60 172.16.0.5
45 192.168.1.50
30 10.0.0.7
```

Meaning:

* `192.168.1.10` made **120 requests**

---

## ⭐ Follow-Up Questions (Very Common)

Interviewers may ask:

**1️⃣ How to find the top endpoint instead of IP?**

```bash
awk '{print $7}' app.log | sort | uniq -c | sort -nr | head
```

**2️⃣ How to find only 500 errors?**

```bash
grep " 500 " app.log
```

**3️⃣ How to count total requests?**

```bash
wc -l app.log
```
