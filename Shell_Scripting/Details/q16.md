## ✅ Coding Question #15 — Top 5 Most Used Commands in Shell History

### Question

Write a **Linux command** to find the **top 5 most frequently used commands** from a user's shell history.

Example source:

```text
~/.bash_history
```

---

## 💡 What the interviewer is testing

They want to see if you understand:

* `history`
* `awk`
* `sort`
* `uniq`
* command pipelines

This checks your **Linux command-line proficiency**.

---

# 🧪 Solution

```bash
history | awk '{print $2}' | sort | uniq -c | sort -nr | head -5
```

---

# 🔎 Explanation

| Command            | Purpose                   |
| ------------------ | ------------------------- |
| `history`          | Shows command history     |
| `awk '{print $2}'` | Extract command name      |
| `sort`             | Group similar commands    |
| `uniq -c`          | Count occurrences         |
| `sort -nr`         | Sort by highest frequency |
| `head -5`          | Show top 5                |

---

# Example Output

```
120 kubectl
95 git
80 terraform
60 ls
40 docker
```

Meaning:

* `kubectl` used **120 times**
* `git` used **95 times**

---

# ⭐ Alternative (Using `.bash_history`)

```bash
awk '{print $1}' ~/.bash_history | sort | uniq -c | sort -nr | head
```

---

# ⭐ Follow-up Questions Interviewers Ask

### ❓ How to clear history?

```bash
history -c
```

---

### ❓ How to search history?

```bash
history | grep kubectl
```

---

### ❓ How to run a previous command quickly?

Press:

```
Ctrl + R
```

Then type part of the command.

---

# 🧠 Real DevOps Insight

Command history analysis can help:

* audit usage patterns
* detect risky commands
* analyze operator activity

Some organizations even **log shell history for security auditing**.
