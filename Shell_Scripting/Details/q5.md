## ✅ Coding Question #5 — Check if a Service is Running (Classic DevOps Question)

### Question

Write a **shell script** to:

* Check if a **service (e.g., nginx)** is running
* If it is **not running**, start the service
* Print a message indicating the status

---

## 💡 What the interviewer is testing

They want to see if you know:

* `systemctl`
* `if` conditions in shell
* service monitoring logic
* basic automation

This is **very common in DevOps/SRE roles** because many automation scripts check service health.

---

## 🧪 Example Script

```bash id="zj3v9m"
#!/bin/bash

service_name="nginx"

if systemctl is-active --quiet $service_name
then
    echo "$service_name is running"
else
    echo "$service_name is not running. Starting service..."
    systemctl start $service_name
fi
```

---

## 🔎 How It Works

| Part                  | Purpose                          |
| --------------------- | -------------------------------- |
| `systemctl is-active` | Checks if the service is running |
| `--quiet`             | Suppresses extra output          |
| `if` condition        | Checks status                    |
| `systemctl start`     | Starts the service               |

---

## 📌 Example Output

If service is running:

```
nginx is running
```

If service is stopped:

```
nginx is not running. Starting service...
```

---

## ⭐ Real SRE Improvement

In production scripts we usually **verify after starting**:

```bash id="bbrh11"
systemctl start nginx

if systemctl is-active --quiet nginx
then
    echo "Service started successfully"
else
    echo "Failed to start service"
fi
```

---

## ⚡ Follow-up Questions Interviewers Ask

### ❓ How to restart the service instead?

```bash id="z9n3qq"
systemctl restart nginx
```

---

### ❓ How to check service status?

```bash id="t09mnh"
systemctl status nginx
```

---

### ❓ How to enable service at boot?

```bash id="ag9e3f"
systemctl enable nginx
```

---

## 🧠 Real Production Scenario

This logic is used in:
