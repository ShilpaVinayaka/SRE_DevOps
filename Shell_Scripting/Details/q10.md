## ✅ Coding Question #10 — Count Lines, Words, and Characters in a File

### Question

Write a **Linux command or script** to count:

* Number of **lines**
* Number of **words**
* Number of **characters**

in a file.

Example file:

```text
app.log
```

---

## 💡 What the interviewer is testing

They want to see if you know:

* `wc` command
* basic **file analysis**
* log inspection tools

This command is used frequently when analyzing **large logs**.

---

# 🧪 Simple Solution

```bash
wc app.log
```

---

## Example Output

```
1200  8500  62000 app.log
```

Meaning:

| Value | Meaning    |
| ----- | ---------- |
| 1200  | Lines      |
| 8500  | Words      |
| 62000 | Characters |

---

# 🧪 More Clear Version

You can specify each metric.

### Count lines

```bash
wc -l app.log
```

---

### Count words

```bash
wc -w app.log
```

---

### Count characters

```bash
wc -c app.log
```

---

# 🧪 Script Version (Interview Friendly)

```bash
#!/bin/bash

file="app.log"

echo "Lines: $(wc -l < $file)"
echo "Words: $(wc -w < $file)"
echo "Characters: $(wc -c < $file)"
```

---

# Example Output

```
Lines: 1200
Words: 8500
Characters: 62000
```

---

# ⭐ Follow-up Questions Interviewers Ask

### ❓ How to count number of ERROR logs?

```bash
grep -c "ERROR" app.log
```

---

### ❓ How to count requests from a specific IP?

```bash
grep "192.168.1.10" app.log | wc -l
```

---

### ❓ How to count number of files in a directory?

```bash
ls | wc -l
```

or safer:

```bash
find . -type f | wc -l
```

---

# 🧠 Real SRE Scenario

This is useful when:

* counting **error occurrences**
* measuring **log growth**
* analyzing **traffic volume**
* debugging **production incidents**
