## ✅ Coding Question #12 — Monitor Disk Usage and Alert

### Question

Write a **shell script** to:

* Check **disk usage**
* If disk usage is **greater than 80%**
* Print an **alert message**

---

## 💡 What the interviewer is testing

They want to see if you know:

* `df`
* parsing command output
* shell conditions
* basic monitoring automation

This is a **very realistic SRE monitoring task**.

---

# 🧪 Step 1 — Check Disk Usage

```bash
df -h
```

Example output:

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   42G   6G   85% /
```

Here:

```
Use% = 85%
```

---

# 🧪 Script Solution

```bash
#!/bin/bash

threshold=80

usage=$(df / | tail -1 | awk '{print $5}' | sed 's/%//')

if [ $usage -gt $threshold ]
then
    echo "ALERT: Disk usage is ${usage}%"
else
    echo "Disk usage normal: ${usage}%"
fi
```

---

# 🔎 Explanation

| Part               | Meaning                       |
| ------------------ | ----------------------------- |
| `df /`             | Disk usage of root filesystem |
| `tail -1`          | Skip header                   |
| `awk '{print $5}'` | Extract usage percentage      |
| `sed 's/%//'`      | Remove `%` symbol             |

---

# Example Output

If disk is high:

```
ALERT: Disk usage is 87%
```

If normal:

```
Disk usage normal: 45%
```

---

# ⭐ Real SRE Improvements

Production scripts usually:

### 1️⃣ Check all filesystems

```bash
df -h | grep '^/dev'
```

---

### 2️⃣ Send alert message

Example:

```bash
echo "Disk usage high" | mail -s "Disk Alert" admin@example.com
```

---

### 3️⃣ Run every 5 minutes

Using **cron**

```
*/5 * * * * /scripts/disk_check.sh
```

---

# 🧠 Real Production Scenario

Disk alerts are triggered when:

* logs grow too large
* container logs fill disk
* backup files accumulate
* temporary files not cleaned

This script is basically a **primitive monitoring system**.
