## ✅ Coding Question #4 — Find Largest Files (Disk Troubleshooting)

### Question

Your **server disk is full**.
Write a **Linux command** to find the **10 largest files in the system**.

---

## 💡 What the interviewer is testing

They want to check if you know:

* `du`
* `sort`
* `head`
* disk troubleshooting commands

This is **very common in real incidents** when servers run out of disk.

---

## 🧪 Simple Solution

```bash
du -ah / | sort -rh | head -10
```

---

## 🔎 Breakdown

| Command    | Purpose                                               |
| ---------- | ----------------------------------------------------- |
| `du -ah /` | Shows disk usage of all files and folders             |
| `sort -rh` | Sort by **size (human readable)** in descending order |
| `head -10` | Show top 10 results                                   |

---

## 📌 Example Output

```
4.5G /var/log/app.log
3.2G /var/lib/docker/overlay2
2.1G /home/user/database_dump.sql
1.9G /var/log/syslog
1.5G /tmp/backup.tar
```

This helps you quickly identify **which files are consuming the most space**.

---

## ⭐ Better Production Command (More Accurate)

```bash
find / -type f -exec du -h {} + | sort -rh | head -10
```

This ensures **only files are listed**, not directories.

---

## ⚠️ Real SRE Tip

Running on entire `/` may show permission errors.

So many engineers use:

```bash
sudo du -ah / 2>/dev/null | sort -rh | head -10
```

`2>/dev/null` → hides permission errors.

---

## Follow-up Questions Interviewers Ask

### ❓ How to check which directory is consuming most space?

```bash
du -sh /*
```

---

### ❓ How to check disk usage of filesystem?

```bash
df -h
```

---

### ❓ How to find large files only in `/var/log`?

```bash
find /var/log -type f -size +100M
```

---

## 🧠 Real Production Scenario

When disk fills up, common causes are:

* log files not rotated
* docker container logs
* core dumps
* backup files
* temp files

This command helps identify them **within seconds**.
