## ✅ Coding Question #14 — Delete Files Older Than X Days (Log Cleanup)

### Question

Write a **shell command/script** to:

* Find files in a directory
* That are **older than 7 days**
* **Delete them automatically**

Example directory:

```text
/var/log/app
```

---

## 💡 What the interviewer is testing

They want to see if you know:

* `find`
* file timestamps
* cleanup automation
* disk maintenance

This is **very common in real production systems** because logs grow quickly.

---

# 🧪 Basic Solution

```bash
find /var/log/app -type f -mtime +7 -delete
```

---

# 🔎 Explanation

| Option              | Meaning                     |
| ------------------- | --------------------------- |
| `find /var/log/app` | Search directory            |
| `-type f`           | Only files                  |
| `-mtime +7`         | Files older than **7 days** |
| `-delete`           | Remove the files            |

---

# Example Scenario

Directory contains:

```
app.log
app.log.1
app.log.2
old_backup.log
```

Files older than **7 days** will be deleted.

---

# ⭐ Safe Approach (Very Important)

Good engineers **check before deleting**.

```bash
find /var/log/app -type f -mtime +7
```

After verifying:

```bash
find /var/log/app -type f -mtime +7 -delete
```

---

# ⭐ Production Version

Often logs are **compressed before deletion**.

Example:

```bash
find /var/log/app -type f -mtime +7 -exec gzip {} \;
```

---

# ⭐ Automate with Cron

Run **daily cleanup**:

```
0 3 * * * find /var/log/app -type f -mtime +7 -delete
```

Runs every day at **3 AM**.

---

# 🧠 Real SRE Scenario

Disk issues in production are often caused by:

* unrotated logs
* large debug logs
* temporary files
* backup files

Many systems rely on **automated cleanup scripts like this**.
