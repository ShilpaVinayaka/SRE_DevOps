## ✅ Coding Question #7 — Find and Kill High Memory Process

### Question

Write a **Linux command or script** to:

* Find the **process consuming the most memory**
* Print the **PID and process name**
* **Kill the process**

---

## 💡 What the interviewer is testing

They want to see if you understand:

* `ps`
* `sort`
* `head`
* `kill`
* basic **process management**

This is a **very common production troubleshooting task**.

---

# 🧪 Step 1 — Find the process using the most memory

```bash
ps aux --sort=-%mem | head -2
```

---

## 🔎 Explanation

| Command        | Purpose                              |
| -------------- | ------------------------------------ |
| `ps aux`       | Lists all running processes          |
| `--sort=-%mem` | Sort by memory usage (highest first) |
| `head -2`      | Shows header + top process           |

---

## Example Output

```
USER  PID  %CPU %MEM COMMAND
root  3456 12.3 40.5 java
```

Here:

* **PID = 3456**
* **Process = java**
* Memory usage = **40.5%**

---

# 🧪 Step 2 — Kill the process

```bash
kill -9 3456
```

`-9` → force kill.

---

# 🧪 One-Liner (Advanced Answer)

Interviewers love this:

```bash
kill -9 $(ps aux --sort=-%mem | awk 'NR==2 {print $2}')
```

---

## 🔎 What happens here

| Part                     | Meaning                        |
| ------------------------ | ------------------------------ |
| `ps aux --sort=-%mem`    | Sort processes by memory usage |
| `awk 'NR==2 {print $2}'` | Get PID of top process         |
| `$(...)`                 | Pass PID to kill command       |

---

# ⚠️ Real SRE Safety Tip

Never immediately `kill -9`.

First check the process:

```bash
ps -fp 3456
```

Then try **graceful kill**:

```bash
kill 3456
```

Only use `kill -9` if it refuses to stop.

---

# ⭐ Follow-up Questions Interviewers Ask

### ❓ How to find top CPU consuming process?

```bash
ps aux --sort=-%cpu | head
```

---

### ❓ How to monitor processes continuously?

```bash
top
```

or

```bash
htop
```

---

### ❓ How to see memory usage of system?

```bash
free -h
```

---

## 🧠 Real Production Scenario

High memory processes are often caused by:

* **memory leaks**
* **bad deployments**
* **infinite loops**
* **unoptimized queries**
* **container memory limits exceeded**

SREs use this technique during **incident debugging**.
