To **execute a shell script in Linux**, there are a few common ways. In interviews, they usually expect you to know **two main methods**.

---

## 1️⃣ Execute using `bash` (most straightforward)

```bash
bash script.sh
```

Example:

```bash
bash cleanup_logs.sh
```

This tells the **bash interpreter** to run the script.

✅ Works even if the script **does not have execute permission**.

---

## 2️⃣ Make the script executable and run it

### Step 1: Give execute permission

```bash
chmod +x script.sh
```

### Step 2: Run it

```bash
./script.sh
```

Example:

```bash
chmod +x cleanup_logs.sh
./cleanup_logs.sh
```

`./` means **run from the current directory**.

---

## 3️⃣ Using shebang (best practice for scripts)

At the top of the script you usually write:

```bash
#!/bin/bash
```

Example script:

```bash
#!/bin/bash
echo "Cleaning old logs..."
```

Then run:

```bash
./script.sh
```

The **shebang tells the system which interpreter to use**.

---

## ⚡ Interview Tip (important)

Interviewers sometimes ask:

**Why do we use `./script.sh` instead of just `script.sh`?**

Because the **current directory (`.`) is not in the PATH variable** for security reasons.

---

## Quick Summary

| Method          | Command              |
| --------------- | -------------------- |
| Run with bash   | `bash script.sh`     |
| Make executable | `chmod +x script.sh` |
| Execute         | `./script.sh`        |
