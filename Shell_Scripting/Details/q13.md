## ✅ Coding Question #13 — Auto-Restart a Process if It Crashes

### Question

Write a **shell script** that:

* Checks if a **process is running**
* If the process **is not running**
* **Restart the process**

Example process:

```text
my_app
```

---

## 💡 What the interviewer is testing

They want to see if you understand:

* `ps`
* `grep`
* process monitoring
* simple **self-healing automation**

This is a **very real SRE concept** — systems should **auto-recover**.

---

# 🧪 Script Solution

```bash
#!/bin/bash

process="my_app"

if pgrep $process > /dev/null
then
    echo "$process is running"
else
    echo "$process is not running. Restarting..."
    /path/to/my_app &
fi
```

---

# 🔎 Explanation

| Command        | Purpose                    |
| -------------- | -------------------------- |
| `pgrep`        | Checks if process exists   |
| `> /dev/null`  | Hides output               |
| `if` condition | Determines status          |
| `&`            | Runs process in background |

---

# Example Output

Process running:

```
my_app is running
```

Process stopped:

```
my_app is not running. Restarting...
```

---

# ⭐ Follow-up Questions Interviewers Ask

### ❓ How to check process using `ps` instead of `pgrep`?

```bash
ps aux | grep my_app | grep -v grep
```

---

### ❓ How to restart a service instead of a process?

```bash
systemctl restart my_app
```

---

### ❓ How to run this script continuously?

Using **cron**:

```
*/1 * * * * /scripts/process_check.sh
```

Runs every **1 minute**.

---

# 🧠 Real SRE Insight

Modern systems do this automatically using:

* **systemd restart policies**
* **Kubernetes pod restart**
* **Docker restart policy**
* **supervisord**

Example:

```bash
Restart=always
```

in systemd service files.
